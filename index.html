⚠️ תפיסות שפג תוקפן

הפריטים הבאים נתפסו ועדיין לא נמכרו. יש להחליט האם להמשיך לשמור או לשחרר:

📧 מדריך הגדרת מערכת המיילים (EmailJS)

מדריך זה מסביר כיצד לחבר את המערכת לשליחת מיילים אוטומטיים — לדוגמה, תזכורות על "נוסח הנצחה" שלא נמסר עדיין על ידי תורם.

המערכת מבוססת על שירות חינמי בשם EmailJS, המאפשר לשלוח מיילים ישירות מהאתר, ללא צורך בשרת נפרד. התוכנית החינמית מאפשרת כ-200 מיילים בחודש — מספיק בהחלט לתזכורות שוטפות.

שלב 1: יצירת חשבון EmailJS

  1. גשו לאתר: emailjs.com
  2. לחצו על Sign Up ופתחו חשבון חינמי (אפשר עם חשבון Google).
  3. אשרו את כתובת המייל שלכם אם תתבקשו.

שלב 2: חיבור שירות מייל (Email Service)

  1. בלוח הבקרה, פתחו את Email Services.
  2. לחצו Add New Service.
  3. בחרו את הספק המתאים — מומלץ Gmail (הכי פשוט לחבר).
  4. מומלץ ליצור חשבון Gmail ייעודי לעמותה ולהתחבר איתו.
  5. אשרו את ההרשאות המבוקשות.
  6. לאחר החיבור תופיע Service ID — לדוגמה: service_abc1234. שמרו אותו בצד.

שלב 3: יצירת תבנית מייל (Email Template)

  1. בלוח הבקרה, פתחו את Email Templates.
  2. לחצו Create New Template.
  3. הגדירו את שלושת השדות הבאים בדיוק כך:
    • To Email: {{to_email}}
    • Subject: {{subject}}
    • Content (תוכן ההודעה): {{message}}
  4. אפשר לעצב את גוף המייל כרצונכם (לוגו, כותרת, חתימה) — רק חשוב שהתוכן הדינמי יופיע במשתנה {{message}}.
  5. שמרו את התבנית. תופיע Template ID — לדוגמה: template_xyz789. שמרו אותו בצד.

שלב 4: איתור המפתח הציבורי (Public Key)

  1. בלוח הבקרה, פתחו Account ← General.
  2. אתרו את Public Key — לדוגמה: AbCdEfGhIjKlMnOpQ. שמרו אותו בצד.

שלב 5: הזנת הפרטים במערכת ניהול ההנצחות

בעמוד הגדרות זה, בקלף "📧 הגדרות EmailJS" מלאו:

  • Service ID — מה שקיבלתם בשלב 2
  • Template ID — מה שקיבלתם בשלב 3
  • Public Key — מה שקיבלתם בשלב 4
  • תבנית תזכורת (ימים) — כמה ימים לאחר מכירה לפני שליחת תזכורת (מומלץ: 14)
  • נוסח המייל — טקסט חופשי, אפשר להשתמש במשתנים: {{comm_name}} (שם ההנצחה), {{project_name}} (שם הפרויקט), {{donor_name}} (שם התורם), {{sale_date}} (תאריך המכירה)

לחצו "שמור הגדרות מייל".

שלב 6: הגדרת כתובות מייל לפי מקור תרומה

בקלף "✉️ מיילים לפי מקור תרומה" — לכל מקור תרומה הזינו את כתובת המייל של האחראי שיקבל את התזכורות עבור אותו מקור, ולחצו "שמור מיילים".

איך זה עובד בשטח

כאשר אדמין מתחבר למערכת, היא בודקת אוטומטית אם יש הנצחות שנמכרו ועדיין "ממתינות לנוסח" מעבר למספר הימים שהוגדר, ומציגה התראה. בעמוד "דוח מקורות תרומה" יש כפתור "✉️ שלח תזכורות נוסח" — לחיצה עליו שולחת מייל לכל אחראי מקור שיש לו הנצחה ממתינה.

בדיקה שהכל עובד

  1. ודאו שיש לפחות הנצחה אחת "שנמכרה" עם "נוסח ממתין" ועם מקור תרומה שהוגדרה לו כתובת מייל.
  2. גשו ל"דוח מקורות תרומה" ולחצו "✉️ שלח תזכורות נוסח".
  3. בדקו שהתיבה מקבלת מייל.

הערה

המדריך הזה מתייחס רק לתזכורות נוסח הנשלחות לאחראי מקור. שליחת מייל אישי לתורם (כפתור "📧 שלח מייל לתורם" בכרטיס ההנצחה) פועלת כרגע באמצעות פתיחת תוכנת המייל הרגילה במחשב — לא דרך EmailJS.

בעיות נפוצות

  • "נא להגדיר EmailJS בהגדרות המערכת" — סימן שאחד השדות (Service ID / Template ID / Public Key) ריק. ודאו שכולם הוזנו ונשמרו.
  • המייל לא מתקבל — בדקו תיקיית ספאם, וודאו שה-Service ב-EmailJS עדיין מחובר (לעיתים Gmail מבקש אישור מחדש כל כמה חודשים).
  • חרגתם מהמכסה החינמית (200 מיילים בחודש) — אפשר לשדרג לתוכנית בתשלום בחשבון ה-EmailJS.
